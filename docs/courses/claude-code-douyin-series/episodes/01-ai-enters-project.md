# 01 我让 Claude Code 随便看看，它差点开始上班

## 成片标题

我让 Claude Code 随便看看，它差点开始上班

## 封面文案

这不是聊天框，是坐到你电脑前的人

## 成片脚本

我第一次用 Claude Code，真的没想太多。

我就打开项目，打了一句：

“帮我看看这个项目。”

我以为它会像 ChatGPT 一样，问我：“你想看哪块？”

结果它不问。

它先看 README。

我说，行，合理。

它又看 package.json。

我说，也合理。

它接着看 docs、src、测试目录、配置文件。

我手已经从键盘上拿开了。

然后它问了一句：

“要不要运行这个命令？”

我当时的感觉，就是请了个朋友来家里看电脑。

我说你帮我看看怎么回事。

他点点头，先把冰箱打开了。

不是说他坏。

是他太进入状态了。

这就是 Claude Code 跟普通聊天 AI 最大的区别。

ChatGPT 更像你电话里问一个懂技术的朋友。你得把代码截图、报错、目录结构一点点发过去。

Claude Code 不一样。

它就在项目里。

它能读文件，搜代码，改文件，跑命令，看命令输出，再决定下一步干什么。

所以它不是“更聪明的聊天框”。

它更像一个能动手的助理。

好处很明显。

你不用来回复制粘贴。

它能自己找线索。

它能跑测试。

它能看结果。

坏处也很明显。

它真能动手。

普通 AI 说错了，最多让你心烦。

Claude Code 说错了，可能顺手给你改了三个文件。

所以第一次用它，最接地气的建议就一句：

别在太大的目录里打开。

别在 `Documents` 这种地方开。

你以为它在看一个项目。

它可能看到五个项目、两个旧 demo、一个你三年前写到一半的工具，还有一堆你自己都不想承认的文件夹。

在哪个目录启动，它就围着哪个目录干活。

这事特别像带新人进公司。

你把他带到会议室，他看白板。

你把他带到工位，他看代码。

你把他带到老板办公室，他可能开始问年终奖怎么算。

Claude Code 也有权限确认。

它改文件、跑命令前，一般会问你。

但别把这个当保险。

它问“能不能运行命令”，不代表这个命令就该运行。

这就像别人问你：“我能不能动一下你家总闸？”

他确实很礼貌。

但你最好先问他为什么。

这一集你只要记住三件事就够了。

第一，Claude Code 能进项目，不是只能聊天。

第二，它靠工具干活，读文件、搜代码、跑命令都算工具。

第三，刚开始先让它只读看看，别一上来就改。

我后来第一次进陌生项目，都会先这样说：

“先别改代码，也别运行会改状态的命令。你先看一圈，告诉我这个项目是干什么的、怎么启动、怎么测试、哪些文件别碰。”

这句话很朴素。

但能省很多事。

因为 Claude Code 最容易出事的时候，不是它不会。

是它太会了。

## 拍摄梗点

- “帮我看看项目”变成“它开始上班”。
- 朋友修电脑先打开冰箱。
- 在 `Documents` 启动看到五个项目和旧 demo。
- 新人进老板办公室问年终奖。
- 权限确认像问“能不能动你家总闸”。

## 知识点藏法

- Claude Code vs ChatGPT：从“电话朋友”和“坐在电脑前的人”讲。
- 代理循环：读文件、找线索、跑命令、看结果、继续下一步。
- 工具调用：用“它真能动手”解释风险。
- 安装入口和目录权限：轻讲，重点落在“在哪打开”。
- 权限模式：不讲太满，只先埋“它会问，但你得判断”。

## 屏幕动作

- 打开一个项目根目录，不打开上级目录。
- 让 Claude Code 只读浏览 README、package.json、docs。
- 展示一次命令权限确认。
- 屏幕大字卡：`先看，不改。`

## 可抄提示词

```text
先不要改代码，也不要运行会改变状态的命令。
请只读看一遍当前项目，告诉我：
1. 这个项目是干什么的；
2. 怎么启动、测试、构建；
3. 关键目录和入口文件在哪里；
4. 哪些文件可能涉及密钥、生产配置或敏感数据；
5. 你的判断依据来自哪些文件。
```

## 结尾衔接

下一集讲我第二个坑。我没让它随便看了，我说“帮我加个功能”。然后它真的加了，问题就出在它真的加了。
