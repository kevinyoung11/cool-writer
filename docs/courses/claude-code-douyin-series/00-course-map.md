# Claude Code 抖音课程总纲

## 一句话卖点

别再找“神提示词”了。Claude Code 真正厉害的地方，不是替你写一段代码，而是能进入项目、调查现场、制定计划、调用工具、验证结果，最后把经验沉淀成自动化工作流。

## 叙事主线

6 集共用一个隐藏主线：

> 一个开发者把 Claude Code 当成更聪明的聊天框使用，边用边踩坑。每一集讲一个真实使用场景，最后搭出一套能长期用的流程。

项目实践弱化为“案发现场证据”。只在这些时候出现：

- 需要解释 Claude Code 能读项目、改文件、跑命令时。
- 需要展示 Plan Mode、`CLAUDE.md`、测试、Hooks、MCP、Subagents、Skills 时。
- 需要说明“真实闭环”和“看起来完成”有什么区别时。

## 六集结构

| 集数 | 故事事故 | 核心知识点 |
|---:|---|---|
| 01 | 用户说“随便看看”，AI 已经准备开始干活 | Claude Code vs Claude Chat、安装入口、目录权限、代理循环、工具调用、权限模式 |
| 02 | 加个暗黑模式，结果页面像停电了 | 第一条提示词、Plan Mode、Explore -> Plan -> Code -> Verify -> Commit、验收证据 |
| 03 | 昨天会启动，今天又问一遍 | `CLAUDE.md`、上下文窗口、压缩、文件引用、上下文冲突、开新会话 |
| 04 | 三个子代理像三个群聊 | Subagents 是什么、什么时候用、如何设计、如何创建、什么时候不要用 |
| 05 | 复制同一段提示词复制到像快捷键 | Skills 是什么、如何创建、多文件配置、排障 Skill、共享 Skill、和其他功能的区别 |
| 06 | AI 写得快，但流程要跟上 | MCP、Hooks、权限、自动化验证、功能组合、长期 AI 开发系统 |

## 每集节奏

```text
0-5 秒：一句离谱事故
5-25 秒：把事故说具体
25-90 秒：调查事故原因
90-210 秒：不断升级事故，同时拆知识点
210-260 秒：给判断标准和可抄模板
260-300 秒：回收开场包袱，接下一集
```

## 口播标准

- 每集保留几个能记住的小梗，但别为了搞笑硬塞。
- 技术点尽量从实际操作里带出来，不要突然转成讲义。
- 每个核心概念配一个日常比喻或真实失败场景。
- 结尾简单收住，接下一集，不上价值。

## 视觉建议

- 封面像“技术事故通报”，例如：`AI 接手项目第一天，把自己当老板了`。
- 录屏不要一直展示代码，穿插大字卡和命令输出。
- 关键术语每集只重点打 2-3 个：`Plan Mode`、`Context`、`Skill` 等。
- 项目画面作为证据，不作为主角。

## 统一口播风格

- 不说“大家好今天我们来学习”。
- 先说场景：“我让 Claude Code 随便看看，结果它已经准备开始干活。”
- 多用连续追问：“它为什么敢改？它凭什么知道？它真的知道吗？”
- 用反差解释概念：“它不是不会写，是太会写了，所以更危险。”
- 技术点尽量回收成一句人话。

## 统一行动号召

- 1-2 集：评论区回复“提示词”，领取第一条 Claude Code 提示词模板。
- 3-4 集：评论区回复“上下文”，领取 `CLAUDE.md` 和 Subagent 模板。
- 5-6 集：评论区回复“工作流”，领取 Skills / Hooks / MCP 组合清单。
