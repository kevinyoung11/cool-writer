## 3. 总体写作定位

数字生命卡兹克的内容定位更像：

> 站在 AI 技术变化的第一现场，用普通人能听懂的语言，把工具、趋势、案例、焦虑和行动建议揉成一篇“可传播的技术观察”。

它不是纯新闻，不是纯教程，也不是纯观点评论，而是几种形态的混合：

1. **技术新闻入口**：抓住一个新模型、新产品、新概念、新事件。
2. **创作者第一视角**：不是旁观者，而是“我试了、我做了、我踩了、我判断”。
3. **社会化解释**：把技术变化翻译成普通人的职业、创作、效率、信任、软件、组织变化。
4. **强情绪标题**：标题常带有冲击、规模、反常识或时代判断。
5. **方法论收束**：最终不是停在新闻，而是落到“你现在该怎么理解/怎么用/怎么行动”。

## 4. 常见选题结构

### 4.1 大事件型

例子：GPT-4o、Sora、Claude、Kimi、微信 Agent、AI 软件吞噬等。

结构通常是：

```text
新事件爆发 -> 为什么这次不一样 -> 技术/产品变化在哪里 -> 对行业/个人意味着什么 -> 我的判断
```

这类文章的重点不是复述新闻，而是解释“这件事改变了哪条规则”。

### 4.2 实操案例型

例子：用 AI 做《流浪地球3》概念预告片、AI Skills、数字员工、日记、Loop Engineering。

结构通常是：

```text
我/别人做了一个东西 -> 过程数字化展示 -> 中间遇到的问题 -> 为什么它代表趋势 -> 普通人可以怎么学
```

这类文章的传播点是“看得见的结果 + 具体过程数字”。比如“5 个晚上、693 张图、185 个镜头、60 个镜头”这类量化细节，会让案例显得真实、可复述。

### 4.3 观点方法论型

例子：AI 时代人才特质、超级个体、三年 AI 使用心得、AI 如何重塑写作与信任。

结构通常是：

```text
先承认焦虑/混乱 -> 给出核心判断 -> 拆成若干条原则 -> 每条配现象/案例 -> 回到个人行动
```

这类文章不是“教工具”，而是“给读者一套解释世界的框架”。

## 5. 标题方法

从公开索引看，标题常见几种模式：

### 5.1 数字 + 奇观

例如：

- 150 万个 Clawdbot 挤爆了一个 AI 论坛
- 花 5 个晚上用 AI 自制《流浪地球3》
- 3 年干了 600 篇 AI 文章

方法：用数字先制造可信度和规模感，再用奇观制造点击理由。

可复用公式：

```text
[数字/规模] + [反常识事件] + [读者熟悉对象]
```

### 5.2 技术人格化/仪式化

例如：

- GPT-4o 的葬礼
- AI 正在吞噬所有软件
- 现实不存在了

方法：把抽象技术变化拟人化、戏剧化，让读者先感到“这事有冲击”。

可复用公式：

```text
[技术名词] + [人类社会动作/强情绪动词]
```

### 5.3 趋势判断 + 行动焦虑

例如：

- AI 时代，为什么我极力推荐你开始写日记？
- 超级个体时代，如何用 AI 打造个人核心竞争力？
- 未来属于 Loop Engineering

方法：先把技术变化上升为时代变化，再给读者一个“我该不该行动”的入口。

可复用公式：

```text
[时代变化] + [你为什么要做某件事]
```

## 6. 开头设计

轻量样本显示，他的开头大概率不是传统科普的“定义式开头”，而更接近以下几种：

### 6.1 事件现场开头

先把读者扔到一个刚发生的 AI 事件里：某模型发布、某产品爆了、某社区被挤爆、某视频刷屏。

作用：降低理解门槛。读者不用先懂理论，只需要知道“发生了个事”。

### 6.2 个人经验开头

用“我试了/我用了/我观察到/我之前一直想”进入。

作用：建立实践者身份，而不是纯评论员身份。

### 6.3 反常识判断开头

先给一个略夸张但有解释空间的判断：现实不存在了、AI 正在吞噬软件、写日记变重要、某模型像葬礼。

作用：让读者带着疑问往下读。

可复用开头模板：

```text
昨天/最近发生了一件很小但很不对劲的事。
表面上它只是 [事件]。
但我越看越觉得，它真正改变的是 [更大的规则]。
这篇文章想讲清楚三件事：
第一，它到底发生了什么；
第二，为什么这次不一样；
第三，普通人现在该怎么做。
```

## 7. 正文推进方式

### 7.1 从现象到机制

先讲“发生了什么”，再讲“为什么会发生”。

这比直接讲理论更适合公众号读者，因为技术读者和非技术读者都能先进入场景。

结构：

```text
现象 -> 过程 -> 机制 -> 影响
```

### 7.2 从案例到时代判断

很多文章不是只停留在工具演示，而是把单个案例推到更大的判断：AI 生成视频不是某个软件升级，而是内容生产流程变化；AI Agent 不是插件，而是操作系统级入口变化。

结构：

```text
具体案例 -> 抽象变量 -> 行业变化 -> 个人策略
```

### 7.3 多段短节奏推进

从摘要和二次描述看，他的文章适合移动端阅读，可能更常用短段落、强转折、口语化判断，而不是长论文式段落。

建议仿写时保持：

- 一段只讲一个意思；
- 多用“但问题是”“真正关键的是”“这意味着”；
- 每隔几段给一个小结论；
- 理论不要连续铺太久，及时回到案例。

## 8. 案例展示方法

### 8.1 案例要有过程数字

轻量索引里最强的案例是 AI 版《流浪地球3》：5 个晚上、693 张图、185 个镜头、60 个镜头。这种数字不是装饰，而是可信度。

可复用方式：

```text
我用了多久；
生成了多少素材；
筛掉了多少；
最终留下多少；
中间失败在哪里；
最后效果如何。
```

### 8.2 案例要连接趋势

案例不是“我很厉害”，而是“这个案例说明门槛变了”。

例如 AI 视频案例真正要表达的不是某个视频多炫，而是个人创作者能用新工具进入过去需要团队/预算的生产环节。

### 8.3 案例要保留粗糙感

如果案例太完美，反而像广告。更好的方式是展示：哪里不稳定、哪里失败、哪里成本高、哪里还需要人工。

这样读者会觉得真实，也更容易相信后面的判断。

## 9. 理论表达方法

他的理论表达适合概括成：

> 不先讲定义，先讲变化；不先堆概念，先讲它改变了谁的行为。

常见理论表达路径：

```text
技术变化 -> 信息流转变化 -> 组织/职业/创作变化 -> 个体策略变化
```

比如：

- AI 软件吞噬所有软件：不是讲“软件定义”，而是讲交互入口和工作流被模型重组。
- 超级个体：不是讲创业学概念，而是讲一个人借助 AI 获得过去团队级能力。
- 日记/写作：不是讲文学，而是讲 AI 时代个人记忆、上下文和信任资产。

可复用理论句式：

```text
每一次技术变化，表面上改变的是 [工具]，本质上改变的是 [信息/成本/协作/分发]。
所以真正受影响的不是 [表层对象]，而是 [更底层的行为方式]。
```

## 10. 风格特征

### 10.1 口语化但不浅

公开样本里有明显口语表达，如“这玩意”“用 AI 的这三年”“该来的还是会来”。这类表达降低阅读门槛，但选题本身并不浅。

### 10.2 强判断但留行动出口

标题和摘要常有强判断，但文章最终通常会回到行动建议：如何使用、如何理解、普通人怎么办。

### 10.3 技术乐观 + 现实焦虑

不是单纯吹 AI，也不是单纯唱衰。更像一种“兴奋但警觉”的语气：AI 很强，但变化太快；机会很多，但不行动就会被甩开。

### 10.4 创作者身份强

他不是只报道 AI，而是反复展示自己作为使用者、创作者、实践者的体验。这使文章更像“从现场发回来的观察”，不是二手新闻整理。

## 11. 结尾设计

结尾通常可以有三种收法：

### 11.1 行动建议型

把全文判断落到读者可以做的事：开始写日记、开始用工具、开始建立技能、开始实践。

### 11.2 时代判断型

用一句更大的判断收束，比如“未来属于某类人/某种工程方式/某种组织方式”。

### 11.3 余味型

不给完整答案，留一个未来感或焦虑感，让读者觉得这件事还在继续发生。

可复用结尾模板：

```text
所以这件事真正重要的，不是 [表层事件]。
而是它提醒我们：
[旧规则] 正在失效，
[新规则] 正在形成。
你不需要立刻成为专家，
但你至少要开始亲手试一次。
因为这个时代最危险的，不是工具不够强，
而是你一直站在旁边看。
```

## 12. 可复用写作方法论：卡兹克式 AI 技术公众号文章

### Step 1：选一个“正在发生”的 AI 事件

不要从抽象主题开始。先找一个有现场感的触发器：模型发布、产品爆火、社区事故、创作者案例、工具更新、争议事件。

### Step 2：给它一个强判断标题

标题要同时满足三件事：

- 有事件；
- 有判断；
- 有读者利益或时代焦虑。

### Step 3：开头先讲现场，不讲定义

第一屏不要解释概念，先让读者知道“发生了什么”和“为什么不寻常”。

### Step 4：用数字和过程建立可信度

尽量给出时间、数量、成本、步骤、失败次数、筛选过程。

### Step 5：把案例抽象成变量

问：这个案例改变的到底是什么？成本、门槛、速度、角色、协作方式、分发方式，还是信任结构？

### Step 6：从变量推出趋势

不要直接说“这是未来”。要说：因为某个变量变了，所以某些旧规则会失效，某些新机会会出现。

### Step 7：给普通人行动建议

最后必须回答：读者现在能做什么？

可以是一个工具、一个习惯、一个判断标准、一个练习、一个避坑建议。

## 13. 适合你借鉴到视频脚本里的点

结合你现在要做 AI/vibe coding 方法论视频，最值得借鉴的是：

1. **用个人实践做入口**：不是“AI 正在改变世界”，而是“我用 AI 做这个项目时，发现了一个问题”。
2. **多用过程数字**：多少视频、多少任务、多少次失败、多少版本、多少测试通过。
3. **把工具上升到行为变化**：Claude Code 不只是写代码，而是改变问题解决吞吐。
4. **每个理论都配一个真实场景**：token、双模型、多线程、AI 过滤，都要配你的项目故事。
5. **结尾落行动**：给观众一套 prompt / 工作流，而不是只给观点。

## 14. 下一步建议

如果后续要升级成“中量版”，建议补三类正文样本：

1. 明确来自其公众号的 10 篇原文链接；
2. 3 篇爆款事件型文章；
3. 3 篇方法论型文章；
4. 3 篇工具/案例型文章。

拿到正文后，可以进一步做：段落长度统计、开头 300 字结构、结尾 300 字结构、案例密度、理论句式、转场词库、标题公式库。
